## 🔧 **Problem-Solving Methodology**

### **Step 1: Problem Analysis**
- **Reading Carefully**: Understand all requirements
- **Identifying Constraints**: Note input/output limits
- **Understanding Examples**: Analyze sample cases
- **Clarifying Ambiguities**: Resolve unclear points

### **Step 2: Algorithm Selection**
- **Pattern Recognition**: Identify problem type
- **Algorithm Choice**: Select appropriate algorithm
- **Complexity Analysis**: Evaluate time/space complexity
- **Alternative Approaches**: Consider multiple solutions

### **Step 3: Implementation Strategy**
- **Planning**: Outline implementation steps
- **Data Structure Choice**: Select optimal structures
- **Code Structure**: Plan code organization
- **Testing Strategy**: Plan testing approach

### **Step 4: Implementation**
- **Clean Coding**: Write readable, efficient code
- **Modular Design**: Use functions and classes
- **Error Handling**: Include error checking
- **Documentation**: Add comments where needed

### **Step 5: Testing and Debugging**
- **Sample Testing**: Test with provided samples
- **Edge Cases**: Test boundary conditions
- **Performance Testing**: Test with large inputs
- **Debugging**: Systematic error finding
