## Tier 1: Essential (Use Every Week)

These are the resources you MUST know and use regularly. No other resources can substitute.

### USACO Guide
**URL**: https://usaco.guide/
**What**: The authoritative preparation guide for USACO, maintained by USACO finalists and IOI team members.
**Content**: Structured curriculum for Bronze → Silver → Gold → Platinum with curated problem lists, editorials, and module-by-module learning path.
**Why essential**: This is built by the people who actually compete. Problems are tagged by division and topic. Every USACO past problem has a difficulty rating and editorial link.
**Use**: Follow the curriculum module by module. Solve all "Easy" and "Normal" problems per module, solve "Hard" problems as stretch goals.

### CSES Problem Set
**URL**: https://cses.fi/problemset/
**What**: 300 canonical competitive programming problems curated by Antti Laaksonen (author of the CP Handbook).
**Content**: Introductory → Sorting and Searching → Dynamic Programming → Graph Algorithms → Range Queries → Tree Algorithms → Mathematics → String Algorithms → Geometry → Advanced Techniques
**Why essential**: These 300 problems cover every fundamental competitive programming topic. Solving all of them puts you firmly in Gold territory and approaching Platinum. Unlike Codeforces where quality is mixed, every CSES problem is carefully chosen.
**Use**: Complete sections in parallel with your USACO Guide curriculum. If you can solve all 300 CSES problems, you are ready for Gold division.
**Link to editorial hints**: https://cses.fi/problemset/list/ (each problem has a hint system)

### Competitive Programmer's Handbook (CPH)
**URL**: https://cses.fi/book/ (free PDF)
**Author**: Antti Laaksonen
**What**: Free 300-page textbook covering all competitive programming topics from basics to advanced.
**Content**: Time complexity → Sorting → Data structures → Graph algorithms → DP → Tree algorithms → Math → String algorithms → Geometry → Advanced topics
**Why essential**: Free, comprehensive, well-written. Better than most paid textbooks for competitive programming.
**Use**: Read alongside USACO Guide; CPH gives the theoretical explanation while USACO Guide gives the practice problems.

### CP-Algorithms
**URL**: https://cp-algorithms.com/
**What**: Comprehensive algorithm reference with implementations and mathematical proofs.
**Content**: Algebra → Data structures → DP → Graph theory → String processing → Linear algebra → Combinatorics → Geometry → Misc
**Why essential**: Best reference for looking up an algorithm's exact implementation. Every major CP algorithm is here with explanation and C++ code.
**Use**: Reference when you encounter a new algorithm; don't read cover-to-cover.

## Tier 2: Practice Platforms

### USACO Official
**URL**: https://usaco.org/
**What**: Official USACO contest platform with complete archive of all past problems.
**Use**: Practice on past contest problems (you can submit to the judge for feedback). Contest archive goes back to 2012.
**Key page**: https://usaco.org/index.php?page=contests — all contests listed by year

### Codeforces
**URL**: https://codeforces.com/
**What**: The most active competitive programming platform globally. Hosts 2–4 rated contests per week.
**Content**: ~8000 problems from Div. 3 (beginner) to Div. 1 (expert). Educational rounds with editorials.
**Use by division**:
- Bronze level: Div. 3 A/B problems, Div. 2 A problems (rating 800–1200)
- Silver level: Div. 2 B/C/D problems (rating 1200–1800)
- Gold level: Div. 1 B/C or Div. 2 D/E problems (rating 1800–2400)
- Platinum level: Div. 1 C/D/E (rating 2400–3200)
**Rating system**: Participate in rounds to build a rating (Newbie → Pupil → Specialist → Expert → Candidate Master → Master → Grandmaster). Targets: Silver = Specialist (1400+), Gold = Expert (1900+), Platinum = Master (2100+).
**EDU section**: https://codeforces.com/edu/courses — structured lessons on specific topics (segment trees, DSU, etc.)

### AtCoder
**URL**: https://atcoder.jp/
**What**: Japanese competitive programming platform with extremely high-quality problems.
**Contest types**:
- **ABC** (AtCoder Beginner Contest): 6–8 problems, A–F/G. A-C = easy, D-E = Silver/Gold level, F-G = Platinum level
- **ARC** (AtCoder Regular Contest): 6 problems, harder. C-E = Gold/Platinum level
- **AGC** (AtCoder Grand Contest): 6 problems. C/D/E = IOI level
**Key resources**:
- [AtCoder DP Contest](https://atcoder.jp/contests/dp) — 26 canonical DP problems, must-solve
- [Kenkoooo AtCoder Problem Tracker](https://kenkoooo.com/atcoder/) — track your progress, do virtual contests
**Rating system**: AtCoder rating 1200+ = Silver, 1600+ = Gold, 2000+ = Platinum, 2400+ = IOI level

### oj.uz
**URL**: https://oj.uz/
**What**: Online judge hosting IOI problems + other major olympiad problems (CEOI, COCI, JOI, Baltic OI, Balkan OI, etc.)
**Why critical for Platinum/IOI**: Has ALL past IOI problems from 2010–present with a judge. Also has problems from ~15 regional olympiads not available elsewhere.
**Key feature**: Subtask scoring — see exactly which subtasks you pass, just like real IOI.
**Use**: After reaching solid Platinum, solve IOI problems from 2018–2024 in chronological order. For each, attempt before reading editorial.

### Kenkoooo AtCoder Problems
**URL**: https://kenkoooo.com/atcoder/
**What**: Unofficial AtCoder problem browser + contest simulator.
**Features**:
- Filter problems by difficulty rating
- Virtual contests: simulate a contest with past problems
- Track your progress across all AtCoder problems
**Use**: Set up virtual AtCoder contests to practice under timed conditions. Filter to your current difficulty range.

### Virtual Judge (VJudge)
**URL**: https://vjudge.net/
**What**: Aggregates problems from 30+ online judges. Create contests with problems from any source.
**Use**: For training groups — create a shared contest with problems from USACO, Codeforces, AtCoder, etc.

## Tier 3: University Courses

### MIT 6.006 Introduction to Algorithms
**URL**: https://ocw.mit.edu/courses/6-006-introduction-to-algorithms-fall-2011/
**Level**: Bronze → Silver
**Topics**: Sorting (merge sort, heapsort), searching, hashing, graphs (BFS, DFS, Dijkstra, Bellman-Ford), DP
**Materials**: Full lecture videos + notes + problem sets
**Use**: Watch lectures for topics you find unclear. Better theoretical foundation than tutorials.

**URL**: https://ocw.mit.edu/courses/6-046j-design-and-analysis-of-algorithms-spring-2015/

**Contest**: Alexandria, Egypt
**Problems + editorial**: Available on ioi.org and oj.uz
**Key problems**: Nile, Message, Mosaic, Tree, Sphinx's Riddle

### Competitive Programming Reference for IOI Language
```
IOI allowed languages: C++17, Java 17, Python 3 (but C++ is effectively required)
Memory limit: usually 512 MB or 1 GB
Time limit: usually 2s (C++), 6s (Java), 10s (Python)
Python is essentially unusable for any non-trivial IOI problem.
```

## Tools and Development Environment

### VS Code Setup for Competitive Programming
```json
// settings.json — add to VS Code settings
{
    "C_Cpp.default.cppStandard": "c++17",
    "code-runner.runInTerminal": true,
    "code-runner.executorMap": {
        "cpp": "cd $dir && g++ -O2 -o $fileNameWithoutExt $fileName && ./$fileNameWithoutExt"
    }
}
```

### Compile Flags (use these)
```bash
# For debugging
g++ -std=c++17 -O2 -Wall -Wextra -fsanitize=address,undefined -o solution solution.cpp

# For submission (maximum optimization)
g++ -std=c++17 -O2 -o solution solution.cpp
```

### Competitive Programming Editors
- **VS Code** with Competitive Companion extension (auto-fetches test cases from Codeforces/AtCoder)
- **CLion** (JetBrains, paid — free for students)
- **Vim/Neovim** with CP plugins (popular among high-level competitors)

### Online IDEs
- **OnlineGDB**: https://www.onlinegdb.com/ (debugging, gdb integrated)

- **Codeforces IDE**: built into every contest
